1.1计算机网络基本概念

1.1.1、什么是计算机网络?

  • 计算机网络就是==通信技术==与==计算机技术==紧密结合的产物
  • 通信系统模型
  • 计算机网络就是一种特殊的通信网络（即信源和信宿就通常是计算机）
  • 定义：计算机网络就是==互连==的、==自治==的计算机集合。自治-无助关系，互连-互联互通。
  • 距离远数量达，如何保证互连
  • 什么是Internet？-组成细节角度
    全球最大的==互联网络==ISP（Internet Service Provider）
    • 计算设备集合：主机，互联网应用等
    • 通信链路：光纤，铜缆等
    • 分组交换：路由器，交换机等
  • 什么是Internet？-服务角度
    • 为网络应用提供通信服务的通信基础设施:Web,email，网络游戏等
    • 为网络应用提供应用编程接口（API）

1.1.2、什么是网络协议？

  • 计算机网络中的==数据交换==必须遵守事先约定好的==规则==。如同交通规则。
  • 定义：为进行网络中的数据交换而建立的规则、标准或约定。
  • 协议规定了通信实体之间所交换的信息的==格式==、==意义==、==顺序==以及针对收到的信息或发生的事件所采取的“==动作==”（actions）
  • 协议三要素：1·语法（Syntax） 2·语义(Semantics) 3·时序(Timing)
  • 协议规范了网络中所有==信息发送==和==接受过程==。

1.2计算机网络结构

1.2.1、计算机网络结构

  • 网络边缘
    • 主机：运行网络应用程序Web，email
    • 客户端/服务器
    • P2P：通信在==对等==实体间直接进行，QQ、Skype
  • 接入网络，物理介质
    • Q:如何将网络边缘接入核心网（边缘路由器）？ A：接入网络（家庭，机构，无线）
    • 家庭接入网络1：数字用户线路（DSL） ==电话线==
    • 家庭接入网络2：电缆网络 ==电视线==
    • 企业接入网络1：以太网构建局域网
    • 无线接入网络
    • 用户关心：带宽 与 共享/独占
  • 网络核心
    • 网络核心：互联的路由器构建的网络
    • 网络核心的关键功能：==路由==+==转发==
    • 网络核心解决的基本问题：Q，如何实现数据从源主机通过网络核心送达目的主机？A,数据交换

1.2.2、Internet结构

  • 端系统通过access ISPs接入Internet
  • 接入ISP必须进一步互连 IXP(Internet exchange ponit)

1.3网络核心

网络核心解决的基本问题：Q，如何实现数据从==源主机==通过网络核心送达==目的主机==？A,==数据交换==

1.3.1、数据交换之电路交换

  • 最典型电话交换网络：电话网络
    • 建立连接（呼叫）
    • 通信
    • 释放连接（挂机）
    • 特点：==独占资源==
  • 多路复用技术
    • 是通信技术的基本概念
    • 链路/网络资源划分为“==资源片==”，将资源片分配给各路“呼叫”（calls），每路呼叫==独占==分配到的资源片进行通信，资源片可能“闲置”（idle）.
  • 多路复用之频分多路复用FDM ：各个用户占用不同频率带宽资源，通信过程至始至终就占用这个频率
  • 多路复用之时分多路服用TDM :将时间划分等的==时间复用帧==（TDM帧），每个TDM帧又划分为多个时隙，每个用户在所占的时隙==周期性交换数据==，周期就是TDM帧。
  • 多路复用之波分多路复用WDM ：就是光的频分复用
  • 多路复用之码分多路复用CDM
    个人多人

1.3.2、数据交换之报文交换

  • 报文：源发用信息整体，比如一个文件，报文交换就是把整个报文一次性发送。
  • 典型的应用：电报的发送

1.3.3、数据交换之分组交换

  • ==分组==：报文拆分成一系列相对较小的数据包
  • 数据包需要在数据前加上头
  • 分组交换需要报文的==拆分==与==重组==
  • 产生==额外的开销==
  • 分组交换：==统计多路复用==
  • ==存储-转发==
    • 报文交换与分组交换均采用存储-转发交换方方式。
    • 区别：报文交换以完整==报文==进行存储转发；分组交换以较小的==分组==进行存储转发

1.3.4、数据交换——报文丶分组交换比较

  • 传输时延

传输数据时时一个比特一个比特

  • 报文交换 vs 数据交换?


  • 分组交换优于报文交换
    • 分组交换传输是==并行==的，报文交换是==串行==的；所以时间短
    • 路由器的==缓存==更低
    • 现代的数据交换技术都是采用分组交换
  • 分组交换的报文交付时间

1.3.5、数据交换——电路丶分组交换比较

  • 考研例题：

  • 分组交换 vs 电路交换
  • 分组交换绝对优于电路交换？
    • 分组交换：适用于==突发==数据传输网络,==非突然==有可能会出现卡顿的问题，如：浏览网页，发邮件等；非突发：看视频，电话
    • 分组交换优点：资源充分共享；简单无需呼叫建立
    • 分组交换缺点：可能产生拥塞，即分组延迟和丢失
